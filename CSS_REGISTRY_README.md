# CSS Registry System Implementation

## Overview

The CSS Registry system has been implemented to optimize CSS generation by collecting and outputting only the CSS that is actually used on a given page. This drastically reduces the amount of CSS generated by excluding unused style definitions.

## Key Components

### 1. CSS_Registry Class (`/inc/KadenceBlocks/Frontend/CSS_Registry.php`)

The main registry class that:
- Tracks used global style slugs and preset slugs
- Collects CSS rules organized by category (global_styles, presets, blocks, components, media_queries)
- Generates optimized CSS based on only the styles actually used on the page
- Provides deduplication of CSS rules

### 2. CSS_Engine Integration

The CSS_Engine class has been updated with:
- A `set_registry_mode()` method to enable/disable registry usage
- Integration points to register global styles and presets with the registry
- Support for outputting CSS to the registry instead of directly to static arrays

### 3. Abstract_Block Methods

New protected methods added to Abstract_Block:
- `register_block_css()` - Register block-specific CSS
- `register_component_css()` - Register component CSS
- `register_preset_css()` - Register preset styles

### 4. Global_Style_Css Integration

The Global_Style_Css class now:
- Registers global style mappings with the CSS Registry
- Registers component presets with the CSS Registry
- Allows the registry to track which styles are actually used

## How It Works

1. **Collection Phase**: As blocks are rendered, they register their used global styles and presets with the CSS Registry
2. **CSS Generation**: The registry generates CSS only for the registered styles
3. **Output**: The optimized CSS is output in the `wp_head` action

## Usage Example

```php
// Enable registry mode in CSS Engine
$css_engine = new CSS_Engine();
$css_engine->set_registry_mode( true );

// Register a global style
$registry = CSS_Registry::get_instance();
$registry->register_global_style( 'my-style-id', $mapping_data );

// Register a preset
$registry->register_preset( 'typography', 'hero-text', $preset_data );

// Register block CSS
$registry->register_css( $css_rules, 'blocks', 'my-block-id' );

// Output the optimized CSS
$registry->output_css();
```

## Benefits

1. **Reduced CSS Size**: Only outputs CSS for styles actually used on the page
2. **Better Performance**: Less CSS to parse and apply
3. **Maintainability**: Centralized CSS management
4. **Debugging**: Easier to track which styles are being used
5. **Future-Ready**: Foundation for critical CSS extraction and other optimizations

## Migration Path

The system maintains full backward compatibility. To enable the registry system:

1. Update blocks to use `$css_engine->set_registry_mode( true )` when generating CSS
2. The registry will automatically track and optimize CSS output
3. No changes required to existing block implementations
